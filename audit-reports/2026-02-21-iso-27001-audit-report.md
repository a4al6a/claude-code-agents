This repository is a collection of Claude Code agent definitions (Markdown prompt templates), Python calculation libraries, and a Bash installer. Assessed against ISO 27001:2022 Annex A controls relevant to software development and information security, the repository demonstrates significant non-conformities across organizational controls (A.5), technological controls (A.8), and documentation requirements. No ISMS documentation, risk assessment, Statement of Applicability, or Information Security Policy exists. The codebase lacks fundamental security controls including access management, logging, encryption, vulnerability management, change control, and incident response procedures. The Python libraries themselves are well-structured with pure math operations and no external dependencies, but the surrounding development practices, governance, and security infrastructure are absent.

### FINDING-001: No ISMS Documentation Suite

**Severity**: Critical
**Confidence**: High
**Risk Score**: 100 (Severity 5 x Likelihood 5 x Impact 4)
**Location**: Not found (missing control) -- no ISMS scope, SoA, Information Security Policy, Risk Assessment, or Risk Treatment Plan
**Framework Mapping**: ISO 27001:2022 Clauses 4.3, 5.2, 6.1.2, 6.1.3, 8.2, 8.3 -- ISMS Scope, Policy, Risk Assessment, Risk Treatment

**Evidence**:
Glob searches for `SECURITY.md`, `SECURITY`, `security-policy*`, `incident-response*`, `CONTRIBUTING.md`, `LICENSE*`, and any policy or governance documentation returned zero results. The repository contains only a `README.md` focused on usage documentation and agent descriptions. No risk assessment, Statement of Applicability, or information security policy of any kind exists.

**Criteria**:
ISO 27001:2022 requires as a minimum: (1) ISMS scope document (Clause 4.3), (2) Information Security Policy (Clause 5.2), (3) Risk Assessment methodology and results (Clauses 6.1.2, 8.2), (4) Risk Treatment Plan (Clauses 6.1.3, 8.3), (5) Statement of Applicability (Clause 6.1.3). These are mandatory documentation requirements for any ISO 27001 certification. In a Stage 1 audit, their absence would prevent progression to Stage 2.

**Impact**:
Complete absence of ISMS documentation constitutes a major non-conformity that would halt any ISO 27001 certification process at Stage 1. Without these documents, there is no formal basis for information security governance.

**Recommendation**:
Develop the required ISMS documentation suite. For a repository of this nature, a lightweight approach could include: (1) a `SECURITY.md` defining the security policy, (2) a risk register identifying key risks (supply chain, credential leakage, agent prompt injection), (3) a Statement of Applicability mapping relevant Annex A controls, (4) a risk treatment plan with mitigating actions.

### FINDING-002: No Access Rights Management (A.8.3)

**Severity**: High
**Confidence**: High
**Risk Score**: 48 (Severity 4 x Likelihood 3 x Impact 4)
**Location**: Git history -- no branch protection, no CODEOWNERS, single-author direct commits to main
**Framework Mapping**: ISO 27001:2022 A.8.3 -- Access Rights Management (previously A.9.2.2)

**Evidence**:
Git history shows 20 commits all directly to `main` by a single author. No `CODEOWNERS` file exists. No branch protection rules are visible in the local configuration. The `.claude/settings.local.json` file grants `Bash(mkdir:*)` permission broadly:
```json
{
  "permissions": {
    "allow": ["Bash(mkdir:*)"],
    "deny": [],
    "ask": []
  }
}
```

No access control lists, role definitions, or permission matrices are documented anywhere.

**Criteria**:
ISO 27001:2022 A.8.3 requires provision, review, modification, and removal of access rights in accordance with the access control policy. For a source code repository, this means defined roles (maintainer, contributor, reviewer), branch protection, and access review procedures.

**Impact**:
Any collaborator with push access could modify any file, including agent definitions and Python libraries, without review or approval. The `.claude/settings.local.json` grants broad file system permissions.

**Recommendation**:
Enable GitHub branch protection on `main`. Define access roles (maintainer, contributor). Create a `CODEOWNERS` file. Review and restrict the broad `Bash(mkdir:*)` permission in `.claude/settings.local.json`.
